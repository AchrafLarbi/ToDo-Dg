Parametres

Envoi d'emails (Brevo recommande, ou Outlook / Office 365)

Recommande : Brevo (gratuit, 300 emails/jour, fonctionne partout). 1) Creez un compte sur brevo.com. 2) Verifiez votre adresse d'envoi dans Expediteurs & IP > Expediteurs (vous recevez un email de confirmation a cliquer). 3) Recuperez vos identifiants SMTP dans SMTP & API > SMTP. 4) Reportez-les ci-dessous : Serveur = smtp-relay.brevo.com  ·  Port = 587  ·  Login SMTP = celui affiche par Brevo (ex. 8a1b2c001@smtp-brevo.com)  ·  Mot de passe / cle SMTP = la « SMTP key » Brevo (pas le mot de passe du compte)  ·  Adresse expediteur (From) = votre adresse verifiee (ex. hm.mokhtari@esi-sba.dz).

Outlook / Office 365 : Serveur = smtp.office365.com, port 587 (ou 465 en SSL). Login SMTP = votre adresse Outlook, mot de passe = un mot de passe d'application, et laissez « Adresse expediteur » vide (elle vaut alors le login).

Erreur 535 « Authentication unsuccessful » ? Le serveur refuse l'authentification. Causes possibles : (1) le mot de passe n'est pas un mot de passe d'application ; (2) l'option « SMTP authentifie » est desactivee pour votre boite -- frequent sur les comptes d'etablissement (esi-sba.dz, universites...). Dans ce cas, demandez a l'administrateur d'activer le SMTP authentifie, ou utilisez un fournisseur d'envoi dedie qui fonctionne partout : Brevo (smtp-relay.brevo.com, port 587), SendGrid (smtp.sendgrid.net, port 587) ou Gmail (smtp.gmail.com, port 465, avec un mot de passe d'application Google). Renseignez simplement le serveur, le port et les identifiants correspondants ci-dessous.

L'email de test est envoye a votre adresse expediteur ({{ settings.sender_email or settings.smtp_user or 'non configuree' }}).
